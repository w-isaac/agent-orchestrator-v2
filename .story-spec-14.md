# AOV-14: Token usage analytics API: per-task/agent/project summaries with success and rework rates

As a platform operator or team lead, I need a usage analytics API that surfaces token consumption and task outcome metrics broken down by project, agent, and task type — so I can understand true cost-per-task, identify expensive or low-quality agents, and optimize context budgets over time. The task_results table already records tokens_used and duration_ms; this story exposes that data as queryable summaries including first-try success rate, rework rate, and context budget utilization.


## Acceptance Criteria
- Given completed task_results records exist, when I call GET /api/usage/summary?project_id=X, then I receive total tokens used, average tokens per task, total cost estimate, first-try success rate, and rework rate for that project
- When I query GET /api/usage/summary?agent_id=X, then the response breaks down token usage and success/rework rates for that specific agent across all projects
- When I query GET /api/usage/summary?task_type=X, then the response returns aggregated metrics grouped by task type
- Given a task required one or more rework cycles (re-queued after validation failure), then its rework_count is reflected correctly in the rework rate calculation
- When I query context budget utilization, the response includes average percentage of the token budget consumed per task dispatch
- All summary endpoints return results within 500ms for datasets up to 100k task_results rows (index-backed aggregation)

## Technical Approach

The story asks for usage analytics (token consumption, cost, success rates, rework rates) broken down by project, agent, and task type. The existing codebase **already tracks all required raw data** across `agent_runs` and `stories` tables — no `task_results` table exists, so all aggregations will be built on top of the existing schema.

**Key mapping from story language to existing schema:**

| Story Concept | Existing Source |
|---|---|
| `tokens_used` | `agent_runs.input_tokens + output_tokens` |
| `duration_ms` | `agent_runs.duration_ms` |
| `cost_estimate` | `agent_runs.cost_usd` |
| `agent_id` | `agent_runs.agent_role` (+ `registered_agents` for metadata) |
| `project_id` | `agent_runs.project_id` → `projects.id` |
| `task_type` | `agent_runs.agent_role` (po, design, architect, engineering, qa, deployment, smoke) |
| `rework_count` | `stories.retry_count` + `stories.qa_bounce_count` |
| `first-try success` | Stories where `retry_count = 0 AND qa_bounce_count = 0 AND status = 'done'` |
| `context_budget` | **New column needed** — `agent_runs.context_budget_tokens` (populated by agent dispatch) |

**Strategy:**
1. Add a single new API router (`/api/usage`) with summary + export endpoints
2. Add one migration to store context budget ceiling per dispatch
3. Build aggregation queries with index-backed WHERE clauses on existing indexed columns
4. Extend the existing `/api/stats` pattern — synchronous SQLite queries in a single Express handler
5. Frontend: 8 new React components + 2 modified components following existing SPA patterns

## Data Model Changes

### Migration 039: `039_usage_analytics.sql`

```sql
-- Context budget tracking per agent dispatch
ALTER TABLE agent_runs ADD COLUMN context_budget_tokens INTEGER;

-- Composite indexes for sub-500ms aggregation on 100k rows
CREATE INDEX IF NOT EXISTS idx_agent_runs_project_role_status
  ON agent_runs(project_id, agent_role, status);

CREATE INDEX IF NOT EXISTS idx_agent_runs_role_finished
  ON agent_runs(agent_role, finished_at);

CREATE INDEX IF NOT EXISTS idx_agent_runs_project_finished
  ON agent_runs(project_id, finished_at);
```

### No New Tables

All metrics derive from existing `agent_runs` + `stories` via aggregation queries. A materialized summary cache is unnecessary — SQLite handles 100k-row aggregations in <100ms with proper indexes.

### Existing Tables Used (read-only)

| Table | Columns Used | Purpose |
|---|---|---|
| `agent_runs` | `project_id`, `agent_role`, `status`, `cost_usd`, `input_tokens`, `output_tokens`, `duration_ms`, `story_id`, `finished_at`, `context_budget_tokens` (new) | Core metrics source |
| `stories` | `id`, `project_id`, `status`, `retry_count`, `qa_bounce_count` | Rework/success rate derivation |
| `projects` | `id`, `name` | Project metadata for breakdowns |
| `registered_agents` | `id`, `agent_name`, `status` | Agent metadata for breakdowns |
