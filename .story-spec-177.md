# AOV-177: Whitelist auto_approve in PATCH /api/projects/:id and add AutoApproveToggle component

As an admin, I need to be able to enable or disable auto-approval for a project so that stories can be automatically approved without manual intervention. The PATCH /api/projects/:id endpoint must accept and persist the auto_approve boolean field (admin-only). The UI must expose an AutoApproveToggle component in the Project Settings Automation section, showing a confirmation modal when enabling but not when disabling. The toggle must reflect the persisted value on page reload and gracefully revert on save errors.


## Acceptance Criteria
- Given an admin user sends PATCH /api/projects/:id with { auto_approve: true }, when the request is processed, then the value is persisted and returned in the response
- Given a non-admin user sends PATCH /api/projects/:id with auto_approve, when the request is processed, then the API returns 403 Forbidden
- Given the AutoApproveToggle is switched to enabled, when the user confirms the modal, then the value is saved and the toggle reflects true on reload; if the save fails, the toggle reverts to false
- Given the AutoApproveToggle is switched to disabled, when the action occurs, then no confirmation modal is shown and the value is immediately saved

# Architecture Specification: Auto-Approve Toggle for Projects

## Technical Approach

Add `auto_approve` boolean column to `projects` table. Whitelist the field in the existing `PATCH /api/projects/:id` handler with admin-only authorization. Build a React `AutoApproveToggle` component with optimistic updates on disable and a confirmation modal (`ConfirmAutoApproveModal`) on enable. Integrate into existing Project Settings page under a new "Automation" section.

## Data Model Changes

**Table:** `projects` (altered)

| Column | Type | Default | Nullable |
|---|---|---|---|
| `auto_approve` | `BOOLEAN` | `FALSE` | `NOT NULL` |

**Migration:** `migrations/NNN_add_auto_approve_to_projects.sql`

```sql
ALTER TABLE projects
  ADD COLUMN auto_approve BOOLEAN NOT NULL DEFAULT FALSE;
```

No index needed (not a query predicate). Rollback: `ALTER TABLE projects DROP COLUMN auto_approve;`.

## API Changes

### `PATCH /api/projects/:id` (modified)

- **Auth:** authenticated user. If request body contains `auto_approve`, require `role === 'admin'` â†’ else `403 Forbidden`.
- **Request body (additive):**
  ```json
  { "auto_approve": true }
  ```
- **Validation:** must be boolean; reject other types with `400`.
- **Response (200):** full project object including `auto_approve`.
  ```json
  { "id": "...", "name": "...", "auto_approve": true, ... }
  ```
- **Errors:** `400` invalid type, `403` non-admin, `404` project missing.

### `GET /api/projects/:id` (unchanged contract, field included)

Response serializer must include `auto_approve` so the UI can hydrate toggle state.
