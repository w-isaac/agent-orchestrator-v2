# AOV-31: Context preview panel: budget controls, artifact toggling, and greedy knapsack auto-packing

As a developer dispatching tasks, I want to set a maximum token budget in the context preview panel, toggle individual artifacts on or off, and have the panel automatically pack artifacts to fit within the budget using a greedy knapsack algorithm. A budget usage bar should show current consumption and warn me when the total exceeds the limit. When I click dispatch, only the selected (active) artifacts are sent.


## Acceptance Criteria
- Given the preview panel, when I set a token budget, then artifacts are auto-packed to fit within the budget using a greedy knapsack algorithm
- Given an artifact, when I toggle it off, then it is excluded from the dispatch payload and the budget usage bar updates accordingly
- Given the budget bar, when the total token count of selected artifacts exceeds the set limit, then a visible warning is displayed
- Given selected artifacts, when I click dispatch, then only the toggled-on artifacts are sent in the dispatch request

# Architecture Spec: Context Preview Panel — Budget Controls, Artifact Toggling & Greedy Knapsack Auto-Packing

---

## Technical Approach

Add a **token budget system** to the existing Context Preview Panel that allows users to:
1. Set a max token budget for dispatch payloads
2. Auto-pack artifacts using a greedy knapsack algorithm (sort by token count descending, greedily include until budget exhausted)
3. Manually toggle individual artifacts on/off
4. See real-time budget consumption via a usage bar with over-budget warnings
5. Dispatch only active (toggled-on) artifacts

**Key decisions:**
- **Greedy knapsack sort key:** Token count descending (largest-first). No "value" signal exists on artifacts today, so this is the pragmatic default. The algorithm is a simple greedy fit, not a full 0/1 knapsack (which would be O(n×W) and overkill here).
- **Budget warning is informational, not blocking.** Users can dispatch over-budget.
- **Auto-pack fires on input change with 300ms debounce.**
- **Artifact list maintains stable sort order** (token count descending) — no reordering on toggle to prevent disorientation.
- **`token_count`** is a new column on the `artifacts` table — populated at artifact creation time by the producing agent.

## Data Model Changes

### Altered Table: `artifacts`

| Column | Type | Default | Notes |
|---|---|---|---|
| `token_count` | `INTEGER` | `NULL` | Estimated token count of the artifact content. Nullable for backward compat with existing rows. |

### Migration: `039_add_token_count_to_artifacts.sql`

```sql
ALTER TABLE artifacts ADD COLUMN token_count INTEGER DEFAULT NULL;

CREATE INDEX IF NOT EXISTS idx_artifacts_token_count
  ON artifacts(story_id, token_count DESC)
  WHERE token_count IS NOT NULL;
```

**No new tables required.** The budget value is ephemeral client-side state — not persisted. Toggle state is transient per-session.

## API Changes

### 1. Modified: `GET /api/stories/:id/artifacts`

**No schema change.** Response already returns all artifact columns. The new `token_count` column will appear automatically. Frontend filters/sorts client-side.

**Response shape** (existing, now includes `token_count`):
```json

  "artifacts": [
    {
      "id": "uuid",
      "story_id": "uuid",
      "stage_id": "uuid|null",
      "type": "string",
      "title": "string",
      "url": "string|null",
      "content": "string|null",
      "token_count": 2048,
      "superseded": 0,
      "superseded_at": null,
      "superseded_reason": null,
      "created_at": "ISO8601"
    }
  ]
}
```

### 2. Modified: `POST /api/stories/:id/artifacts`

Accept optional `token_count` in request body.

| Field | Type | Required | Notes |
|---|---|---|---|
| `token_count` | `integer` | No | Token count of artifact content |

### 3. New: `POST /api/stories/:id/dispatch`

Dispatches a story with a filtered set of artifact IDs (only the toggled-on ones).

**Request:**
```json
{
  "artifact_ids": ["uuid1", "uuid2"],
  "token_budget": 4096
}
```

**Response (200):**
```json
{
  "story_id": "uuid",
  "dispatched": true,
  "artifact_count": 2,
  "total_tokens": 3584,
  "token_budget": 4096
}
```

**Behavior:**
- Validates all `artifact_ids` belong to the story and are not superseded
- Stores the selected artifact IDs + budget in `prompt_context` JSON on the story row
- Transitions story status from `queued` → `queued` (stays queued; scheduler picks it up)
- Broadcasts `story_context_updated` via WebSocket

**Error responses:**
- `400` — empty `artifact_ids` array
- `404` — story not found
- `422` — one or more artifact IDs not found or superseded
