# AOV-32: Project dashboard page with v2 API integration

As a user, I want a project dashboard page that lists all my projects fetched from the v2 API, so I can quickly see project status and task counts at a glance and drill into any project for more detail. The dashboard should display projects as cards with name, status, and task counts, and support a project detail view showing full metadata and a task summary. All data fetching must target /api/v2/ routes exclusively.


## Acceptance Criteria
- Given the dashboard page loads, when the component mounts, then it fetches projects from GET /api/v2/projects and renders each as a card showing name, status, and task counts
- Given a project card is rendered, when the user clicks it, then a project detail view is shown with the project's metadata and a task summary
- Given any data fetch in the dashboard or detail view, when the request is made, then it targets /api/v2/ routes only — no /api/v1/ calls are present

# Architecture Spec: Project Dashboard Page with v2 API Integration

## Technical Approach

Build a React SPA client (`/app/client/`) from scratch using Vite + TypeScript + React Router. Create v2 API routes on the Express backend that reshape existing data from the `projects` and `stories` tables to match the dashboard contract. The v2 endpoints aggregate story counts inline (avoiding N+1) and return a simplified shape optimized for the dashboard UI.

**Key decisions:**
- **No new database tables or migrations** — all needed data exists in `projects` and `stories`
- **v2 routes as a new router module** — keeps v1 untouched, clean separation
- **Client bootstrapped from scratch** — no existing React source files exist; only `client/dist/` is present
- Task counts returned inline on the projects list endpoint (resolves Open Question #1)
- No pagination for now (resolves Open Question #3 — project count is small)
- Single-page summary for detail view, no tabs yet (resolves Open Question #4)
- No "create project" CTA in empty state (resolves Open Question #5 — out of scope)

## Data Model Changes

**No schema changes required.**

Existing tables provide all necessary data:

| Table | Fields Used |
|---|---|
| `projects` | `id`, `name`, `slug`, `status`, `description`, `created_at`, `updated_at` |
| `stories` | `project_id`, `status` (queued/in_progress/gate/done/failed) |

The v2 API will aggregate `stories` counts per project via SQL `GROUP BY` (same pattern already used in `/api/dashboard`).

**Status enum** (from existing data): `active`, `archived` — maps to badge colors per UX spec. If other values appear, default to a neutral badge.

## API Changes

### New: `/api/v2/projects` router

All endpoints mounted under `/api/v2/projects` via a new router file.

#### `GET /api/v2/projects`

Returns all projects with inline task (story) counts.

**Response:**
```json
[
  {
    "id": "uuid",
    "name": "Maple Mana",
    "slug": "maple-mana",
    "status": "active",
    "description": "...",
    "created_at": "ISO8601",
    "updated_at": "ISO8601",
    "task_counts": {
      "total": 12,
      "open": 5,
      "in_progress": 2,
      "complete": 5
    }
  }
]
```

**Implementation:** Single query joining `projects` with aggregated `stories` counts. Maps story statuses: `open` = queued + gate + failed, `in_progress` = in_progress, `complete` = done.

#### `GET /api/v2/projects/:id`

Returns full project metadata + inline task counts.

**Response:**
```json
{
  "id": "uuid",
  "name": "Maple Mana",
  "slug": "maple-mana",
  "status": "active",
  "description": "...",
  "owner": null,
  "created_at": "ISO8601",
  "updated_at": "ISO8601",
  "task_counts": {
    "total": 12,
    "open": 5,
    "in_progress": 2,
    "complete": 5
  }
}
```

#### `GET /api/v2/projects/:id/tasks`

Returns detailed task (story) breakdown for the task summary panel.

**Response:**
```json
{
  "project_id": "uuid",
  "summary": {
    "total": 12,
    "queued": 3,
    "in_progress": 2,
    "gate": 1,
    "done": 5,
    "failed": 1
  },
  "tasks": [
    {
      "id": "uuid",
      "display_id": "MM-1",
      "title": "...",
      "status": "done",
      "priority": "high",
      "created_at": "ISO8601"
    }
  ]
}
```
