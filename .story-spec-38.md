# AOV-38: Context node endpoints, result normalizer, and output validator

As a backend engineer, I need REST endpoints for task-scoped context node retrieval, a result normalizer to canonicalize raw agent output, and an output validator to enforce schema conformance and confidence thresholds — so that agents can query relevant context before dispatch and pipeline results are consistently structured and validated before graph updates are applied.


## Acceptance Criteria
- GET /api/context-nodes returns a list of context nodes, supporting optional query parameters for filtering by type and label
- GET /api/context-nodes/:id returns a single node's detail including all connected edges
- POST /api/context-nodes creates a new context node accepting type, label, content, and staleness_ttl_ms fields and returns the created node
- contextGraph.js exposes an expandFromSeeds(seedNodeIds, depth) method that returns a subgraph of nodes and edges up to the specified depth
- resultNormalizer.js transforms raw agent output into a canonical object with artifacts, relationships, and metadata fields
- outputValidator.js validates a normalized result for schema conformance, scope adherence, and confidence threshold, returning pass/fail with error details
- Unit tests cover context expansion at depths 0, 1, and 2+ with correct node and edge counts
- Unit tests cover resultNormalizer and outputValidator for both passing and failing cases including missing fields, out-of-scope artifacts, and below-threshold confidence

# Architecture Specification: Context Node Endpoints, Result Normalizer, and Output Validator

## Technical Approach

This story adds three backend capabilities to the agent orchestrator:

1. **Context Node CRUD + Graph Expansion** — Two new SQLite tables (`context_nodes`, `context_edges`) with REST endpoints for listing, detail retrieval, and creation. A `contextGraph.js` library module implements `expandFromSeeds(seedNodeIds, depth)` for BFS subgraph traversal.

2. **Result Normalizer** (`resultNormalizer.js`) — A pure-function module that transforms raw agent output (arbitrary JSON) into a canonical shape: `{ artifacts, relationships, metadata }`.

3. **Output Validator** (`outputValidator.js`) — Validates normalized results against a schema contract, checks scope adherence, and enforces a confidence threshold. Returns `{ pass: boolean, errors: [] }`.

All three modules follow existing project conventions: synchronous `better-sqlite3` for data access, Express Router for API routes, UUID primary keys, ISO timestamps, and Vitest for testing.

**Key design decisions:**
- Context nodes/edges are project-scoped (via `project_id` FK) to align with existing multi-project architecture.
- `expandFromSeeds` uses iterative BFS (not recursive) to prevent stack overflow on deep graphs, with a hard cap of `depth <= 5`.
- Normalizer and validator are stateless library functions (no DB dependency), enabling easy unit testing and reuse inside agent pipelines.
- Frontend components live alongside existing client source; since client source is not in this repo (only `client/dist/`), the spec defines component contracts and props but implementation is deferred to the frontend build pipeline.
