# AOV-42: Context preview: budget usage bar with over-budget warning

As a user dispatching tasks from the context preview panel, I want to see a visual budget usage bar that reflects the total tokens of my selected artifacts against my configured budget limit, so I can make informed decisions about which artifacts to include before dispatching and avoid accidentally sending an oversized context.


## Acceptance Criteria
- Given a token budget is set and artifacts are toggled on, when the panel renders, then a budget usage bar displays showing (total tokens of toggled-on artifacts) / (budget limit) as a filled progress bar
- Given artifacts are toggled on or off, when the toggle state changes, then the budget usage bar updates in real time to reflect the new total
- Given the total tokens of selected artifacts is within the budget limit, when the bar renders, then it is displayed in a non-warning color (e.g. green or neutral)
- Given the total tokens of selected artifacts exceeds the budget limit, when the bar renders, then it turns red and an over-budget warning message is shown
- Given the dispatch button is clicked, when the request is sent, then only toggled-on artifacts are included in the payload
- Unit tests cover: toggle state changes update token totals, budget calculation is correct, over-budget threshold triggers warning display, under-budget state shows no warning

# Architecture Spec: Budget Usage Bar with Over-Budget Warning

**Story:** Context preview panel budget usage bar
**Date:** 2026-04-16

---

## Technical Approach

Add a **client-side budget usage bar** to the Context Preview Panel that visually represents `selectedTokens / budgetLimit`. The bar updates in real time as artifacts are toggled on/off. When over budget, the bar turns red and an advisory warning appears. The dispatch button remains enabled regardless.

**Key decisions:**
- `token_count` is **pre-computed and stored** on each artifact row (avoids on-the-fly computation and loading states)
- `budget_limit` is a **per-story persistent setting** stored in the `stories` table (not session-scoped)
- All budget calculation is **client-side** — no new server-side computation needed beyond exposing the data
- The budget section is **hidden entirely** when no budget is configured (`budget_limit IS NULL` or `0`)

---

## Data Model Changes

### Altered Tables

| Table | Column | Type | Default | Purpose |
|---|---|---|---|---|
| `artifacts` | `token_count` | `INTEGER` | `0` | Pre-computed token count for the artifact content |
| `stories` | `budget_limit` | `INTEGER` | `NULL` | User-configured token budget for context dispatch. `NULL` = no budget set |

### Migration: `039_budget_usage_bar.sql`

```sql
-- Add token_count to artifacts for budget calculation
ALTER TABLE artifacts ADD COLUMN token_count INTEGER NOT NULL DEFAULT 0;

-- Add budget_limit to stories for per-story context budget
ALTER TABLE stories ADD COLUMN budget_limit INTEGER DEFAULT NULL;

-- Index for fast artifact token aggregation per story
CREATE INDEX IF NOT EXISTS idx_artifacts_story_token_count
  ON artifacts(story_id, superseded, token_count);
```

### Backfill consideration

Existing artifacts will have `token_count = 0`. A one-time backfill script should estimate token counts for existing artifact content (e.g., `Math.ceil(content.length / 4)` as a rough chars-to-tokens heuristic). This can be a separate migration or admin script.

## API Changes

### Modified Endpoints

#### `GET /api/stories/:id/artifacts`

**Current response:** Array of artifact objects
**Change:** Each artifact object now includes `token_count: number`

```jsonc
// Response shape (each element)
{
  "id": "uuid",
  "story_id": "uuid",
  "type": "design",
  "title": "UX Design Spec",
  "content": "...",
  "token_count": 4200,       // NEW
  "superseded": 0,
  "created_at": "2026-04-16T..."
}
```

#### `POST /api/stories/:id/artifacts`

**Change:** Accept optional `token_count` in request body. Agents should compute and pass token count when creating artifacts.

```jsonc
// Request body addition
{ "token_count": 4200 }  // optional, defaults to 0
```

#### `GET /api/stories/:id/context`

**Change:** Include `budget_limit` in response.

```jsonc
// Response shape
{
  "story_id": "uuid",
  "dispatched": false,
  "context": { ... },
  "budget_limit": 20000    // NEW — null if not configured
}
```

### New Endpoint

#### `PATCH /api/stories/:id/budget-limit`

Set or update the token budget for a story.

| Field | Details |
|---|---|
| **Method** | `PATCH` |
| **Path** | `/api/stories/:id/budget-limit` |
| **Request body** | `{ "budget_limit": 20000 }` — `null` to clear |
| **Response** | `{ "story_id": "uuid", "budget_limit": 20000 }` |
| **Errors** | `404` story not found, `400` if `budget_limit` is not a positive integer or null |
